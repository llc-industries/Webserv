⚙️ Scripts CGI

— Basiques

request_info.py Affiche méthode, query string et body POST (Python) curl -X POST http://localhost:8080/cgi-tests/request_info.py -d "nom=Alice&age=21"
nom: age:
request_info.php Idem en PHP curl -X POST http://localhost:8080/cgi-tests/request_info.php -d "nom=Alice&age=21"
nom: age:
env_dump.py Dump de toutes les variables d'environnement CGI curl http://localhost:8080/cgi-tests/env_dump.py
login.py Test de session et cookies (Bonus) curl -c cookies.txt http://localhost:8080/cgi-bin/login.py?user=admin

— Headers et parsing

custom_headers.py Envoie Set-Cookie + X-Custom-Header dans la réponse CGI curl -i http://localhost:8080/cgi-tests/custom_headers.py
lf_separator.py Utilise \n\n au lieu de \r\n\r\n — teste la robustesse du parser curl http://localhost:8080/cgi-tests/lf_separator.py
cgi_status_404.py Le CGI retourne "Status: 404" — vérifie que le serveur le respecte curl -i http://localhost:8080/cgi-tests/cgi_status_404.py

— Body POST

post_body_check.py Vérifie que stdin reçoit exactement CONTENT_LENGTH octets curl -X POST http://localhost:8080/cgi-tests/post_body_check.py -d "hello world"
binary_upload.py Upload binaire via stdin CGI, écrit dans upload/chunk_result.bin curl -X POST http://localhost:8080/cgi-tests/binary_upload.py --data-binary @fichier.bin

— Erreurs et crash

crash_before_header.py Crash avant d'écrire les headers → doit retourner 500 curl -i http://localhost:8080/cgi-tests/crash_before_header.py
crash_after_header.py Crash après les headers → comportement différent du précédent curl -i http://localhost:8080/cgi-tests/crash_after_header.py

— Timeout et concurrence

timeout_5s.py Attend 5s — ouvre dans 2 onglets pour tester la concurrence curl http://localhost:8080/cgi-tests/timeout_5s.py & curl http://localhost:8080/ &
timeout_5s.php Idem en PHP curl http://localhost:8080/cgi-tests/timeout_5s.php
timeout_10s.py Attend 10s → doit déclencher le timeout CGI du serveur curl http://localhost:8080/cgi-tests/timeout_10s.py
infinite_loop.py Boucle infinie Python → doit être killé par le timeout curl http://localhost:8080/cgi-tests/infinite_loop.py
infinite_loop.php Boucle infinie PHP → idem curl http://localhost:8080/cgi-tests/infinite_loop.php
large_output.php Génère 100 000 lignes — teste la gestion de gros output curl http://localhost:8080/cgi-tests/large_output.php | wc -l
